## Стек

Astro 6 (static output), TypeScript, MDX, деплой Vercel. Node >= 22.12.0.
Контент: Markdown/MDX в src/content/blog/YYYY/MM/.
Дополнительная дистрибуция: Telegram-канал.

## Архитектура

Статический блог на Astro с content collections. Контент на русском языке (`lang="ru"`).

### Content model

Посты хранятся в `src/content/blog/YYYY/MM/` — организация по году и месяцу. Glob-паттерн `**/*.{md,mdx}` подхватывает вложенные папки автоматически.

Схема frontmatter определена в `src/content.config.ts` (Zod):
- `title`, `description`, `pubDate` — обязательные
- `type: 'morning' | 'evening' | 'weekly'` — обязательное
- `tags: string[]` — по умолчанию `[]`
- `heroImage`, `source`, `updatedDate` — опциональные

### Slug и маршрутизация

Файловая структура `YYYY/MM/` используется только для организации. В URL попадает только имя файла (без пути папок) — логика обрезки в `src/pages/blog/[...slug].astro`.

URL: `/blog/2026-03-25-evening-claude-opus/` (не `/blog/2026/03/...`).

### Layout

`BlogPost.astro` используется и для постов, и для статичных страниц (about). Props типизированы как `Partial<CollectionEntry<'blog'>['data']>` с обязательными `title`, `description`, `pubDate` — чтобы about мог не передавать `type` и `tags`.

Компонент `Disclaimer.astro` автоматически добавляется в конце каждого поста через layout — не дублировать в Markdown.

### Соглашения по файлам

- Имя файла поста: `YYYY-MM-DD-type-slug.md`
- Изображения: `public/images/YYYY-MM-DD/`, формат WebP
- Теги: рубрики (`stocks`, `futures`, `crypto`, `commodities`, `forex`, `macro`), формат (`weekly`, `morning`, `evening`), аналитика (`atas`, `order-flow`, `macro-us`, `macro-ru`, `macro-eu`)

### Промпты

Шаблоны для генерации выпусков лежат в `prompts/`: `morning.md`, `evening.md`, `evening-formal.md`, `evening-casual.md`, `weekly.md`, `glossary.md`. Используются вручную при подготовке постов.

## Форматы выпусков

### Утренний пост
**Блог (300–500 слов):**
1. 📊 Обзор — что произошло за ночь, контекст
2. 💱 Курсы ЦБ РФ — USD/RUB, EUR/RUB на сегодня + EUR/USD
3. 📈 Ключевые уровни — на что смотреть, с пояснением логики
4. 🔭 Фокус дня — события, отчёты, данные

**Telegram (150–200 слов):**
Сжатая версия. Только цифры, уровни, события. Без развёрнутого контекста.
В конце — ссылка «Подробнее в блоге →»

### Вечерний пост
**Блог (600–900 слов):**
1. 📊 Обзор дня — sentiment, главный драйвер, что могло случиться но не случилось
2. 💱 Курсы ЦБ РФ — USD/RUB, EUR/RUB на сегодня и на завтра + EUR/USD
3. 📈 Рынки — индексы, commodities, крипта с разбором причин движения
4. 🔥 Главное — ключевая новость дня, разные мнения, альтернативные сценарии
5. 🔭 Прогноз на завтра — конкретные уровни и события для трейдеров и инвесторов
6. 💡 Что мне с этого? — 1–2 предложения, вывод уровнем выше. Только когда тема дня заслуживает обобщения, не каждый день
7. Дисклеймер

**Telegram (250–350 слов):**
Выжимка: цифры, главная новость, 2–3 ключевых тезиса, прогноз.
Без развёрнутой аналитики и альтернативных сценариев.
«Что мне с этого?» — оставляем, если попадает.
В конце — ссылка «Полный разбор в блоге →»

### Воскресный обзор
**Блог (800–1200 слов):**
1. 📊 Итоги недели — что двигало рынки, несбывшиеся сценарии
2. 📈 Что показали объёмы — анализ через ATAS, голоса разных участников
3. 🏦 Макро-блок — ставки (ЦБ, ФРС, ЕЦБ), инфляция, безработица, состояние экономик РФ/US/EU
4. 🔥 Сетап недели — один инструмент, разбор через volume profile
5. 🔭 Мысль на неделю — ключевые события, на что смотреть
6. 💡 Что мне с этого? — обязательно в воскресном выпуске
7. Дисклеймер

**Telegram (400–500 слов):**
Сокращённый обзор: итоги + сетап + мысль на неделю.
Без полного макро-блока и объёмного анализа.
В конце — ссылка «Полный обзор с графиками в блоге →»

## Различия блог vs Telegram

### Блог
- Развёрнутая аналитика, контекст, альтернативные сценарии
- Графики и скриншоты (TradingView, ATAS, Finviz heatmap)
- Цитаты аналитиков и участников рынка
- Несбывшиеся сценарии для глубины
- SEO-заголовки, теги, frontmatter

### Telegram
- Только суть: цифры → вывод → прогноз
- Без графиков в тексте (максимум 1 скриншот если критично)
- Короткие абзацы, 2–3 предложения максимум
- Каждый пост заканчивается ссылкой на блог
- Задача — зацепить и привести в блог за полной версией

## Макро-блок: правила
- В ежедневных постах — только если вышла значимая новость (решение по ставке, jobs report, данные по инфляции)
- В воскресном обзоре — полный срез по трём экономикам (РФ, US, EU)
- Фокус: как данные влияют на рынки и что с этим делать трейдеру
- Ключевые ставки, инфляция, безработица, PMI, рынок недвижимости — по релевантности

- Arkham — ончейн-аналитика, движения кошельков

- MOEX — рублёвые пары, индекс МосБиржи
- ЕЦБ (ecb.europa.eu) — решения по ставкам, протоколы

### Новостной контекст
- Bloomberg, Reuters, WSJ — глобальные рынки
- РБК (rbc.ru) — российский рынок, экономика РФ
- ForexFactory — экономический календарь

### Sentiment
- CNN Fear & Greed Index — акции
- Alternative.me Fear & Greed — крипта

## Git Conventions
- Ветвление: main (stable), feature/* для новых фич, style/* для экспериментов со стилем
- Коммиты: conventional commits на английском (feat:, fix:, docs:, chore:)
- Формат коммита: type(scope): описание — например docs(blog): add evening digest 2026-04-02
- Не коммитить напрямую в main при экспериментах — через feature-ветку и PR
- .env не коммитить, шаблон в .env.example
- PR содержит: Summary (что сделано) и Test plan (как проверить)
